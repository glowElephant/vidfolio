Product Builder 철학

  • AI가 세상을 지배하는 시대. 코드를 짜는 것이 아닌 문제를 정의하고 제품을 만드는 사람이 필요
  • "이 기술을 사용할 줄 아는가?" → "이 문제를 풀 수 있는가?"로 패러다임 전환
  • 상용화된 AI를 최대한 활용하여 가치를 창출하는 인재

AI 활용 실전 사례

  • Unity NPC에 OpenAI API 연동 → 질문답변, 이미지 분석
  • 나레이션 파이프라인: Applio로 음성모델 학습 → API 자동합성 (비용 80% 절감)
  • 아동 발음 영상: Midjourney/Flux → Applio → D-ID/HeyGen 파이프라인 구축
  • OpenClaw + NotebookLM 아침 뉴스 브리핑 자동화
  • Claude Code + tmux 멀티세션 병렬 개발: 메인 에이전트와 실시간 대화하며 서브 에이전트에 태스크 위임
  • 에이전트 간 AI_Language 압축 통신 프로토콜 → 토큰 절감 + 효율적 협업
  • AI 에이전트를 도구가 아닌 팀원으로 활용하는 AI-First 개발 방식

문제 해결 DNA

  • 카드 연출 수정: 연출자가 Inspector에서 직접 수정 가능한 Tool 개발 → 시간 코스트 대폭 절감
  • 나레이션 외주 파이프라인: 외주→프리랜서→검수→수정 의 비효율 → Applio 기반 자체 생성 파이프라인 구축
  • 반복작업 발견 시 Tool 제작을 우선순위로 고려하는 마인드
